## 3. Infrastructure & fixes

- **Disk crisis.** SSD hit 100% (228 GB), which **evicted the iOS device platform** and
  **logged Xcode out** — the real cause behind a cascade of "No Team" / "iOS 26.5 not
  installed" errors, not a project problem. Freed ~36 GB: simulator runtimes (24 GB),
  simulator devices (10 GB), Docker, npm/Xcode caches. "System Data" was mostly Xcode
  simulator cruft. External 1 TB disk is the long-term fix.
- **`CODE_SIGNING_ALLOWED = NO` in the hand-authored pbxproj** made Xcode produce
  **unsigned archives** → Organizer's "No Team Found in Archive". Flipped app + keyboard
  targets to `YES` — the actual fix (looked like a login/team bug for hours).
- **CloudKit provisioning battle**: registered the container, enabled the iCloud
  capability on the App ID via the **ASC API**, associated the container in the portal,
  then learned the App ID + entitlement + container all being right still isn't enough
  headlessly (see learnings).
- **Release tooling**: `whisperio-release` skill + `mobile/WhisperioApp/Scripts/release-testflight.sh`
  (headless archive → manual-signed export → altool upload). Works for non-CloudKit builds.

## 4. Key learnings / gotchas (read before next release)

1. **CloudKit builds MUST go through Xcode Archive → Distribute.** The public ASC API
   and `xcodebuild` cannot generate a provisioning profile that includes the iCloud
   container entitlement (they strip it → the app faults at launch when an account is
   present). Only Xcode's internal `developerservices2` (GUI, logged in) embeds it.
   Headless (`release-testflight.sh`) is fine only for non-CloudKit builds.
2. **CloudKit schema: Development JIT-creates, Production must be deployed.**
   TestFlight/App Store builds use the **Production** environment. SwiftData auto-creates
   the schema only in **Development** (on a Debug write). You must **Deploy Schema Changes
   to Production** in the CloudKit Console. Symptom of skipping it: works in Xcode, "nothing
   syncs" on TestFlight. Deployed schema: `CD_RecordingEntity`, 14 fields, correct indexes.
3. **Seeding the Dev schema without an iOS device in Developer Mode**: run **WhisperioMac
   Debug** on a Mac signed into iCloud — the `#if DEBUG` seed in `WhisperioMacApp` writes one
   record → JIT-creates the schema in Development → then deploy to Production. (macOS needs
   no Developer Mode.)
4. **CloudKit Console web login = your *developer* Apple ID**, which is often a **different
   account** than the one on your devices. The Console's private DB view will look **empty
   even when device↔device sync works** — it's a different account's data. Don't verify sync
   via the Console; test device→device directly.
5. **Universal app (iOS + macOS under one App Store record) must share one bundle id.**
   Adding the macOS platform to the existing Whisperio record forced `WhisperioMac` onto
   `ai.whisperio.mobile` — bonus: it inherits the already-provisioned CloudKit container.
6. Signing keychain: the distribution cert lives in the **`husar` keychain** (password
   `husar`) — answer the codesign prompt with it + "Always Allow".
7. Disk-full evicts the downloadable iOS platform → `xcodebuild -downloadPlatform iOS`.
